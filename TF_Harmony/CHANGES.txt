Changes made to app_working_backup.R and new files
====================================================

NEW FILES
---------

data_loading.R
  All data file reads and preprocessing that was previously at the top of
  app_working_backup.R (lines 1-155 of the original). This includes reading
  harmony tables, DEGs, TF family annotations, PWMs, cell type expression,
  JIT datasets, vertex setup for network graphs, and all column renaming,
  merging, and filtering done at startup. Loaded via source("data_loading.R").
  Later rewritten to load preprocessed RDS files when available (see
  preprocess_data.R below), falling back to raw TSV processing if not.

preprocess_data.R
  One-time script that reads all raw TSV/CSV data files, performs all column
  renaming, merging, filtering, and cleanup, then saves the final objects as
  compressed RDS files in Data/. Run once with `Rscript preprocess_data.R`
  from the TF_Harmony directory after any data file changes.
  RDS files created: dt.rds, narpv.rds, cte.rds, jitr.rds, jits.rds,
  newdt.rds, vertices.rds, ids.rds, allgeneids.rds, idoptions.rds,
  tfswithfamilies.rds, allteds.rds, pwms_processed.rds, pfms.rds,
  phylo_dend.rds.

helpers.R
  Utility functions extracted from between the UI and server sections of the
  original file. Contains:
  - ftestfun: Fisher's exact test for DEG overlap significance
  - findfile: Looks up significant DEGs for a list of TFs from narpv
  - matchtidy: Pairwise comparison of DE result tables, tagging concordant/discordant
  - harmony_hclust: NEW function that replaced four duplicated clustering
    reactive blocks (~120 lines) with a single parameterized function
  Loaded via source("helpers.R").

ui.R
  The full UI definition extracted from app_working_backup.R. Contains the
  navbarPage layout with all three top-level panels (Global Analyses, Pairwise
  Analyses, Target Regulation) and their sub-tabs. Loaded via source("ui.R").


MODIFICATIONS TO app_working_backup.R
--------------------------------------

Structure:
  - Removed ~155 lines of inline data loading, replaced with source("data_loading.R")
  - Removed ~90 lines of helper functions, replaced with source("helpers.R")
  - Removed ~400 lines of UI definition, replaced with source("ui.R")
  - Main file now contains: library imports, source calls, server function, shinyApp()

Bug fixes:
  - Removed duplicate library(gprofiler2) (was loaded twice, lines 3 and 11)
  - Removed library(Cairo) (unused, caused crash on systems without XQuartz)
  - Removed reactiveConsole(TRUE) (debugging tool, not for production)
  - Fixed broken caching in tarfs/matched reactives: exists("tarfs()") always
    returned FALSE because it checked for a literal variable named "tarfs()".
    Replaced with reactiveVal pattern that correctly passes previous results.
  - Added nrow check in matchtidy (helpers.R) to handle the case where all
    pairs are already cached and tempagg is empty.
  - Fixed SVG/PNG mismatch in motif plot: tempfile had .png extension but
    svg() device was used and contentType declared SVG. Changed to .svg.
  - Fixed viridis palette options: "a" and "e" (lowercase) are invalid,
    changed to "A" (magma) and "E" (cividis).
  - Fixed melt.data.table warnings by adding explicit id.vars and measure.vars
    to all three melt.data.table(subgraph()) calls.

Performance:
  - Removed suspendWhenHidden = FALSE on GO plot output. Previously forced
    the gprofiler2 GOST API call on every app load. Now only runs when the
    user visits the GO Terms tab.
  - Changed allgenes selectInput to server-side selectizeInput for the large
    gene list in Target Regulation. Choices are populated server-side instead
    of sending all gene IDs to the browser on load.
  - Deduplicated four clustering reactives (conhclustx, conhclusty, dishclustx,
    dishclusty) into calls to harmony_hclust() in helpers.R. Reduced ~120 lines
    to ~24 lines. Also fixed inconsistency where concordant reactives were
    subsetting dt directly while discordant ones used subdt().
  - Changed relationships and familyproportion render functions to use subdt()
    instead of duplicating the same family/TF subsetting of dt inline. The
    family/TF filtering now runs once in subdt() and is shared by all consumers.
  - Pre-computed phylogenetic tree (MSA -> alignment -> distance -> bionj -> dendrogram)
    once at startup in data_loading.R instead of recomputing it in three separate
    renderPlot blocks (phyloconTanglegram, phylodisTanglegram, tanglegram).
    The static phylo_dend object is now pruned per-render based on selected TFs,
    but the expensive MSA conversion and tree building only happens once.
  - Removed five orphaned tanglegram render outputs (conmotifTanglegram,
    phyloconTanglegram, phylodisTanglegram, dismotifTanglegram, condisTanglegram)
    that were not referenced in the UI. These were replaced by the single
    output$tanglegram with dropdown selectors but never cleaned up. Removing them
    eliminates ~80 lines of dead server code and stops unnecessary reactive
    computations on every TF selection change.
  - Removed orphaned output$motifbrowser renderPlot (~12 lines). Its tab panel
    was previously commented out of the UI, so the render never displayed.

UI improvements:
  - Replaced GO term table: was publish_gosttable() rendering a static ggplot
    image that could not scroll. Now uses DT::renderDT with a proper DataTable
    showing source, term_id, term_name, term_size, and p_values columns.
    Table is searchable, sortable, and paged (25 rows). P-values sort numerically
    and display in scientific notation (e.g. 5.19e-18). Unlisted the p_values
    list column (from multi_query) by taking the min per term.
  - Fixed GO table layout: replaced splitLayout (caused header misalignment and
    search bar clipping) with fluidRow/column in ui.R. Table now renders
    correctly alongside the GO plot at 50/50 width.
  - Replaced all textInput fields for numeric cutoffs with numericInput,
    adding appropriate min/max/step values. Affects: familyharmonycutoff,
    familypvcutoff, familyintersectcutoff, tfregpcutoff, tfreglfccutoff,
    tfdegreecutoff, regpcutoff, reglfccutoff.
  - Extracted the network layout choices (13 layouts repeated 3 times) into
    a single network_layouts variable in ui.R.
  - Fixed hardcoded CSS path (~/Desktop/github/HTT_144TFs/TF_Harmony/www/style.css)
    to use the local www/style.css path.
  - Removed dead motifloader function (had a typo tag$img instead of tags$img,
    was never called).

Bug fixes:
  - Fixed NxTime splitLayout cellWidths summing to 120% (was "65%","55%").
    Changed to "50%","50%".
  - Fixed unsafe 1:length(Tar1) in matchtidy (helpers.R) which produced
    c(1, 0) when Tar1 was empty. Replaced with seq_along() and added an
    early return for fewer than 2 TFs.
  - Fixed motifplot renderImage error handler: tryCatch caught errors but
    returned nothing, causing renderImage to throw its own error. Replaced
    with validate(need()) to show a clean message when no motifs match, and
    removed the silent-fail tryCatch.
  - Fixed gotable erroring with <2 TFs: DT::renderDT does not propagate
    Shiny's silent reactive cancellation, so goout()$result threw an error.
  - Fixed motif comparison erroring with 0 TFs: empty pattern caused
    pfms[grepl(...)] to return an atomic vector. Added req(length(input$TFs) >= 1).
  - Fixed deprecated ggplot2 size parameter in geom_smooth (harmonyplotly).
    Changed to linewidth for ggplot2 3.4+.
  - Fixed kbreaks selectInput returning character string instead of integer
    for tanglegram k_branches/k_labels. Added as.integer() conversion.
  - Fixed local variables shadowing reactive functions in tanglegram render:
    motifupclust/motifdownclust locals renamed to muc/mdc.
  - Removed vestigial self-assignment in subdt reactive (subdt <- dt[...]
    inside the reactive named subdt). Same cleanup previously applied to
    motifhclust/motifupclust/motifdownclust.

    - subgraph() and matched(): "Select at least two TFs for pairwise analysis."
    - targetsubnar(): "Select at least one target gene."
